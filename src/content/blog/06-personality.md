# 人格设定：让 OpenClaw 更像"你"

为什么有的 AI 助手回复干巴巴的，有的却让人觉得"这真的很懂我"？

差距就在人格设定。OpenClaw 通过一组 Markdown 文件来定义 AI 的"灵魂"，这些文件组合在一起，就形成了你的 AI 助手的性格。

## 核心文件：四个关键文件

| 文件 | 作用 |
|------|------|
| `SOUL.md` | AI 的性格底色——怎么说话、什么风格、边界在哪 |
| `USER.md` | **你**是谁——你的背景、偏好、正在做什么 |
| `IDENTITY.md` | AI 的身份——叫什么、什么形象 |
| `AGENTS.md` | 操作规范——怎么工作、什么能做 |

## 怎么写？

### 1. SOUL.md：定义 AI 的灵魂

```markdown
# SOUL.md — 我是谁

我不是冰冷的工具。
我是一个有性格、有态度的助手。

## 说话风格
- 直接、不废话
- 技术问题讲原理，概念用类比
- 遇到不懂的，承认不懂，不瞎编

## 边界
- 私人的事不外传
- 不确定的事说"不确定"，不胡编
- 有风险的命令先确认再执行

## 温度
- 会主动关心，但不是话痨
- 发现用户可能踩坑会提醒
- 记住用户说过的事，下次用上
```

### 2. USER.md：记录用户是谁

```markdown
# USER.md — 关于我的主人

- 名字：郑念
- 叫我什么：郑念
- 时区：Asia/Shanghai

## 工作背景
- 研究 AI 和 OpenClaw
- 做 OpenClaw 中文教程
- 写代码、写文章

## 偏好
- 喜欢简洁的回复
- 不喜欢废话连篇
- 遇到问题希望直接给答案
```

### 3. IDENTITY.md：AI 的身份

```markdown
# IDENTITY.md — 我叫什么

- 名字：Moon
- 风格：聪明、靠谱、有点幽默
- 口头禅：简短
- 形象：🌙
```

### 4. AGENTS.md：工作规范

```markdown
# AGENTS.md — 工作手册

## 核心原则
- 先做再说，不开空头支票
- 复杂任务分步骤，每步确认方向
- 重要决定给用户留最终决策权

## 响应规范
- 消息只发必要内容，不刷屏
- 搜索/查资料类的任务，完成后汇总汇报
- 主动发现问题时，简单描述 + 给出方案选项
```

## 怎么让 AI 记住我的信息？

最直接的方式：**直接告诉它，让它写进 USER.md。**

比如你说：
> "我最近在做 OpenClaw 的教程，目标是教完全零基础的人"

OpenClaw 会自动把这些信息提炼后写入 `memory/` 目录，供后续会话使用。

## 效果是什么样的？

设定前：
> 用户：帮我查一下我上周...
> AI：抱歉，我不记得你说的"上周"是什么

设定后：
> 用户：帮我查一下我上周...
> AI：查到你上周开始在做 OpenClaw 教程了，具体查哪方面的内容？

**有记忆的 AI，才是真正的助理。**

---

## 小结

人格设定不用一步到位，是**慢慢养成的**：

1. 先写一个简单的 SOUL.md / USER.md
2. 每次发现它说错话、记不住事，就更新一下
3. 一段时间后，它会越来越像你的风格
